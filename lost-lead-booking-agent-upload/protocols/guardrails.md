# Guardrails

## Always

- build for a real business outcome
- keep the MVP small
- keep human review for risky steps
- log calls, bookings, and failures
- make fallback behavior clear
- update project memory after major decisions

## Never

- build features because they sound cool
- automate an unclear workflow
- skip the buyer pain
- let the agent make promises the business cannot keep
- send real SMS or calls without approval
- store or expose secrets

## Good Agent Work

Good work is:

- narrow
- testable
- tied to money
- safe by default
- easy for the next agent to continue
