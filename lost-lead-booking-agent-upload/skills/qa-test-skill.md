# QA Test Skill

## Use When

Testing the MVP before a demo or pilot.

## Always Test

- normal booking
- caller outside service area
- caller wants emergency help
- no calendar slots available
- caller asks for human
- SMS send failure
- calendar booking failure
- duplicate call or duplicate lead

## Never

- call the prototype ready after only one happy path
- test with real customer data first
- ignore failed notifications

## Output

```text
Pass:
Fail:
Bugs:
Launch blockers:
Safe to demo:
```
