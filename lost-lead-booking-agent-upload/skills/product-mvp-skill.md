# Product MVP Skill

## Use When

Defining what to build, who it is for, and what counts as success.

## Always

- start with the buyer's money pain
- keep the first version narrow
- define one measurable outcome
- write what we are not building
- connect every feature to booked jobs or saved time

## Never

- build a generic AI assistant
- add dashboard features before the call flow works
- chase multiple industries at once
- use vague metrics like "better experience"

## Output

```text
Buyer:
Pain:
Money loss:
Agent job:
Human review:
MVP:
Success metric:
Not building yet:
```
